Submission Pipeline
① Record JSON assembled
↓ fetch patient AES-256 key
② Record encrypted (AES-256-CBC)
↓ sign ciphertext with RSA-2048 private key
③ Ciphertext + signature POSTed to node
↓ node verifies signature
④ Record mined into block (PoW)
↓ block broadcast to 2 peer nodes
⑤ Decentralised & immutable ✓

⚠ Tamper Detection Demo
Simulate an insider attack and observe automatic detection + recovery

How it works

Clicking "Run Attack" directly mutates a block's data on Node 1 without updating its hash — exactly what a rogue DBA would do. The system immediately detects the broken hash chain and reports the tampered block index. "Recover" restores Node 1 by syncing from honest Node 2.
